## 🧩 Como eu penso

```text
Problema operacional  →  Modelagem de dados  →  API / Automação  →  Dashboard / Ação
        manual                  Postgres            REST · MCP          KPI · Farol
```

- **Arquitetura primeiro** — Clean Architecture, SOLID e separação por camadas (repository · service · API)
- **Multi-tenant by design** — sistemas pensados para múltiplos clientes desde o schema
- **Privacidade & segurança** — redação de PII (LGPD), whitelist de queries, validação de input nas bordas
- **Automação > processo manual** — confiabilidade e menos intervenção humana como métrica de sucesso
- **Observabilidade** — logs, indicadores e farol de aderência para enxergar a operação em tempo real

## 📌 Projetos em destaque

> 🛰️ **Plataforma de Operações / Workforce** — Backend multi-tenant em **Node.js + PostgreSQL** com APIs REST versionadas, dashboards gerenciais de KPI (demanda × realizado × perdas), monitoramento de workforce e farol de aderência por UF/cidade.

> 🤖 **Servidor MCP de Análise Operacional** — Ferramentas de IA (Model Context Protocol) que permitem consultar e diagnosticar dados operacionais via LLM, com schemas validados, modo mock e redação de PII embutida.

> ⚙️ **Automações de Processos** — Workflows com **Google Apps Script e Webhooks** integrando planilhas, mensageria e sistemas internos para eliminar tarefas repetitivas.

> 📱 **Apps Mobile em Flutter** — Aplicações com **Clean Architecture, SOLID, Provider e GetIt**, persistência local em SQLite e foco em manutenibilidade.
